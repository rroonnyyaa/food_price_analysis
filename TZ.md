# Техническое задание на проект: "Food Price Analytics"

## 1. Цель проекта
Создание веб-сервиса для агрегации и анализа цен на продукты питания в различных магазинах. Проект решает проблему поиска оптимальной цены на продуктовую корзину и помогает отслеживать продуктовую инфляцию, предоставляя пользователям инструменты для сравнения стоимости товаров и анализа динамики цен.

## 2. Роли пользователей
* **Гость**: Просмотр каталога товаров, просмотр графиков динамики цен, сравнение текущих цен в разных магазинах.
* **Зарегистрированный пользователь**: Ведение личного списка покупок (Shopping List), получение рекомендаций по выгодным магазинам для своего списка.
* **Администратор**: Управление справочниками товаров и магазинов, загрузка данных (парсинг/импорт).

## 3. Модели данных

### Сущность 1: Product (Товар)
Описывает конкретный продукт, который мы отслеживаем.
* **Поля**: 
    * `name` (Название товара)
    * `category` (ForeignKey на Category)
    * `description` (Описание)
    * `image` (Изображение - опционально)

### Сущность 2: Store (Магазин)
Описывает торговую сеть или конкретный магазин.
* **Поля**:
    * `name` (Название магазина)
    * `url` (Ссылка на сайт магазина)
    * `address` (Адрес - опционально)

### Сущность 3: PriceRecord (Запись о цене)
Хранит исторические данные о цене товара в конкретном магазине на конкретную дату.
* **Поля**:
    * `product` (ForeignKey на Product)
    * `store` (ForeignKey на Store)
    * `price` (Цена в рублях)
    * `date_recorded` (Дата фиксации цены)

### Сущность 4: ShoppingList (Список покупок)
Персональный список товаров пользователя.
* **Поля**:
    * `user` (ForeignKey на User)
    * `products` (ManyToManyField к Product)
    * `created_at` (Дата создания)

## 4. Ключевой функционал (User Stories)
1. **Поиск и сравнение**: Пользователь вводит название продукта ("Молоко") и видит таблицу с ценами на этот товар в разных магазинах (Пятерочка: 80р, Магнит: 75р).
2. **Анализ динамики**: На странице товара пользователь видит график изменения средней цены за последний месяц (помогает понять, сейчас дорого или дешево).
3. **Оптимизация корзины**: Пользователь добавляет 10 товаров в свой список, и система подсказывает, в каком магазине общая стоимость этой корзины будет минимальной.

## 5. Технический стек и интеграции
* **Backend**: Django 4.x, Python 3.x
* **Database**: SQLite (для разработки)
* **Data Analysis**: 
    * `Pandas`: Для расчета средней цены по категории и анализа продуктовой инфляции.
    * `Chart.js`: Для интерактивных графиков на стороне клиента.
* **Frontend**: Tailwind CSS.
